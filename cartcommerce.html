Project Information

An online application service that allows the user to buy products over the internet. Key features are as follows:

  • Searching Products using keywords, category and price range
  • Shopping Cart that lets customers to store items they are interested to purchase
  • Order History with the ability to view purchased items for customers
  • Customizable website theme of users' choices

Responsibilities

  • Programmed the entire website, which has the use of Java Persistent API, Java HTTP servlet, Logging Filter, and Java Database Connectivity
  • Wrote the Java Server Pages that render web pages to client-side
  • Implemented URLs Forwarding feature that redirects the user back to the desired page after login page
  • Implemented Simple Mail Transfer Protocol (SMTP) Java Mail backend that allows the user to receive emails from Cart-Commerce service, in which the user can:
    • Confirm the email address before using additional features, e.g. User Account, Shopping Cart, Check out, Order History, and so on
    • Reset password
    • Confirm the change of a new email address
    • Receive order information right when the purchase is successful

Problems and Solutions

  • This project was created before we have had an understanding of both Software Development Process and Project Management. During the development process, we kept adding features more and more without putting priority to each task into consideration, which led to unstable goals. Our fix before the project deadline was to stop adding unnecessary features so that the project can achieve the required goals.
  • The project duration is limited to 3 months. We could not learn and make everything perfect in one sitting during the university, as there was not enough time. Due to this, we put priority on learning and working with Java Web API the most so that the project meets all requirements.
  • One of my team members had inadequate web programming skills and did not want to participate in any part of the programming. My fix was to let my member choose the tasks they feel happy with the most (Database design, SQL statements, mockups, maintaining documents, etc., were done for this case).
