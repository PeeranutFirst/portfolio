Godot Mega Man

Personal Project (2019-2021)

Mega man is a side-scrolling platformer game where the player character must fight through the levels using Mega Man’s “Mega Buster”—a cannon attached to his arm—to shoot the robotic enemies inhabiting his environment. The goal is to reach the boss room and defeat the boss to complete the level.

Mega Man is one of my first experiment projects I have made in my spare time during the Bachelor's degree in the university. With my curiosity to learn and try things out, platforming genre is the one that draws motivation due to the assets that can be easily found on the internet and the only game that will never go out of the box. This resulted in a Mega Man clone.

The most challenging part during the game-making process is the design patterns for each scenario, such as finite state machine for the characters, singleton classes that allow global variables to be stored, etc. Before the game come out as a Mega Man clone, the project was just a simple platformer prototype where the character can only run and jump. As more mechanics are added, the project gets harder to maintain, which encourages me to solve the problems by learning more.

My aim for this project is to try and implement things that can be easily expanded or reusable later. However, due to how I was new to Godot engine entirely, the project became difficult to expand as the project grows, which led to too many refactor attempts, as most of the issues were the knowledge base, the project structure management and naming conventions. These led to lessons that allowed me to improve the programming quality, write better, cleaner code and easier ways to maintain the game codes and project. I also learned that this technique can be applied when working with a team, or when someone wants to take a look at the code.
